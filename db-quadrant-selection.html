X-Axis — Schema Flexibility
Rigid (left): Schema is defined upfront and enforced — every row must match. Changes require formal migrations. Enables constraints, foreign keys, and optimizer efficiency.

Flexible (right): Schema is optional or dynamic — each document or node can have different fields. Enables rapid evolution but loses enforcement and query predictability.
Y-Axis — Data Connectivity
Highly Connected (top): Data entities are linked — queries must traverse relationships. "Who knows who?" or "Which accounts share a device?" require relationship-aware engines.

Flat / Sequential (bottom): Records are largely independent or ordered by time. Queries are range-based or key-based — no traversal required. Optimized for append-heavy workloads.
