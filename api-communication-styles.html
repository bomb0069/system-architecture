Y-Axis — Contract Strictness (Typed ↔ Flexible)
Strict / Typed (top): Schema is required. Types are enforced at compile time or code-generation time. Client and server contracts are versioned and validated — breaking changes are caught before deployment.
Flexible / Loose (bottom): Schema is optional or implicit. JSON payload can evolve freely. Easy to prototype, but no automatic safety net for breaking changes.
X-Axis — Communication Direction (Pull ↔ Push)
Pull / Request-Response (left): Client always initiates. Server responds, then the connection closes. Simple, stateless, cacheable — but client must poll if it wants updates.
Push / Event-Driven (right): Server sends data to client when something happens — without the client asking. Requires persistent connection or a delivery mechanism. Essential for real-time.
