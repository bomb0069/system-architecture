Y-Axis — Client Thickness (Thick ↔ Thin)
Thick Client (top): Significant logic, processing, or state lives on the client machine. Can work offline. Client may have its own local database. Updates require distributing new client software.
Thin Client (bottom): Client only renders UI and forwards input. All logic lives on the server. Easier to update (just deploy server). Requires network connection. Web browsers are the ultimate thin client.
X-Axis — Tier Count (Few ↔ Many)
Few Tiers (left): Fewer physical/logical layers. Less network overhead. Simpler to deploy and debug. Harder to scale individual layers independently.
Many Tiers (right): Each concern is a separate layer — presentation, logic, cache, data, edge. Each can scale, be secured, and be updated independently. More complexity, more resilience.
